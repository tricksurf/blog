An introduction to Source Engine modification

In this post, I will be walking through some of the basics of starting an internal tool for manipulating and modifying game behavior. This is intended for people with some experience, but being that I myself am still learning I expect there to be some information helpful for a beginner as well.

End Goal

The end goal of this series of projects is going to be a simple internal bunnyhopping script. While it is a simple thing to implement, there are some basic concepts with it that can be applied to later endeavors. We will do this by detouring the function that the game uses to send movement commands, as well as reading networked variables to get the flags of the player.

Networked variables are variables that are used to share information between the client and the server. We can access these ourselves to utilize information about our own player.
